# Deploy
- 개발자의 컴퓨터에서 설치한 패키지, 필요한 파일들을
실제 서비스를 하고 있는 컴퓨터로 옮기는 과정 


0. 가상환경 생성
python -m venv 가상환경명


1. 배포
1) 필요한 패키지를 현재 상태 그대로 파일로 저장합니다.
- 파이썬패키지는 requirements.txt 라는 파일명을 보통 사용합니다.
pip freeze > requirements.txt 

2) 쓰지 않을 불필요한 파일들/남들에게 공개하지 말아야 할 파일을 제거합니다.
- .gitignore 라는 파일에 불필하거나 공개되면 안 될 파일의 리스트를 작성합니다.
- 벌써 .gitignore 없이 git add .을 했다면 
- git rm --cached *
- 이후 git add . -> git commit -m "메시지" -> git push origin main
* 주의사항: 그럼에도 불구하고 이전 커밋에 남아있음


3) 원격 서버가 확인할 수 있는 레퍼지토리에 올립니다.


# 2. 이미 배포된 서비스를 업데이트할 때
1) pip freeze > requirements.txt 로 패키지 업데이트 상황 재작성
2) 숨겨야할 변경사항이 있다면 .env에 작성하고 .gitignore에 추가합니다.


3. share.streamlit.io에 배포하고
웹사이트 주소를 구글시트에 남겨주세요.
